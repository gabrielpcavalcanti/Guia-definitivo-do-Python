# COMERNada

## C - confiável

O isolamento entre as partes gera algo mais seguro. Ao alterar uma das partes, nenhuma outra é afetada.

Se as especificações do método forem alteradas, mas a ligação entre eles não, pode mudar quanto quiser. Não terá problema.

## O - oportuno

Ao dividir tudo em partes, cada uma delas poder ser desenvolvido em paralelo. Não é preciso que uma parte seja desenvolvida antes da outra, pode ser ao mesmo tempo.

Podem ser criadas várias classes de forma paralela. Uma pode ou não interagir com as outras, independente, elas podem ser desenvolvidas em paralelo.

## M - manutenível

Atualizar é mais fácil. Uma pequena alteração vai beneficiar todas as partes relacionadas.

Se aprimorar uma classe, toda as outras vão se beneficiar junto.

## E - extensível

Um sistema não deve ser estático. Tudo deve mudar e crescer para permanecer útil.

Basicamente fazer atualizações no sistema para poder melhorar ou se adaptar as situações.

## R - reutilizável

Classes que foram criadas para um sistema podem ser aproveitados em outros sistemas.

Bibliotaecas, não reevente a roda.

## N - natural

Mais fácil de entender. Maior atenção às funcionalidades do que aos detalhes de implementação.

Não se preocupa com a arquitetura, ela é de fácil entendimento. Importante é são os métodos.

Não importa como o método foi construído e sim que é facil utilizar ele. Da os argumentos de entrada e sabe a saída e possíveis erros.

---

Objeto é uma variável que além de guardar dados podem executar funcionalidades.
