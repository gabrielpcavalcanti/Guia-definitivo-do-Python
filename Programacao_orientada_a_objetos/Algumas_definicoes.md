# Algumas definições e conceitos

## Classe

Um formato a ser seguido sempre que quisermos fazer objetos do mesmo tipo ou que tenham as mesmas características ou que tenham os mesmos comportamentos.

É a definição de uma entidade que possui características (atributos) e ações (métodos).

A classe é a forma e o objeto é o que é criado quando se usa a forma. Algumas características dos objetos podem mudar, mas a  essencia, o esqueleto é o mesmo. Isso é o que a classa nos dá.

### Diagrama UML

Na análise e desenvolvimento de sistemas orientados a objeto, esse diagrama é muito útil porque informa o que toda classe precisa ter.

A primeira parte é o nome da classe, a segunda é as características que a classe (atributos) tem e a terceira é o que a classe pode fazer e o que podem fazer com elas (os métodos).

## instancia

Seguir o padrão que foi definido na classe para poder criar um objeto. É como se fosse a ação de criar um objeto utilizando uma classe.

No momento em que a classe gera, forja, controi um objeto. Isso é a instância. Estou instanciando um objeto a partir de uma classe.

"O objeto é a instância de uma classe".

Classes são projetadas, modeladas para fazerem objetos padronizados.

## Objeto

Coisa material ou abstrata que é feita a partir de uma classe e pode ser descrita por meio de seus métodos, atributos e estado atual.

## Estado

É o conjunto de valores dos atributos ao passar do tempo. O estado pode mudar ou não.
